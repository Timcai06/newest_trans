# 谷歌翻译插件 - 词性差异化显示系统样式规范

## 1. 概述

本样式规范文档定义了谷歌翻译插件中词性差异化显示系统的设计原则、颜色方案和样式参数。该系统为不同词性的单词提供独特的高亮效果和弹窗样式，确保同一词性的单词始终使用一致的视觉风格，提升用户体验和可读性。

## 2. 设计原则

- **一致性**：同一词性的单词在所有场景下使用相同的颜色方案和样式
- **可区分性**：不同词性之间的视觉差异明显，便于快速识别
- **美观性**：采用现代化的渐变色彩方案，提升视觉吸引力
- **可读性**：确保高亮文本的可读性，避免颜色过于刺眼
- **可维护性**：样式规范清晰，便于后续扩展和修改

## 5. 实现说明

### 5.1 高亮单词实现

通过为不同词性的单词添加特定的CSS类（如 `pos-noun`, `pos-verb` 等），结合CSS渐变背景实现差异化显示。所有高亮单词共享基础类 `translated-word-highlight`，确保统一的基础样式和过渡效果。

### 5.2 弹窗实现

翻译弹窗通过动态添加CSS类（如 `pos-noun`, `pos-verb` 等）来应用不同的背景渐变和样式。弹窗样式与对应词性的高亮颜色保持一致的渐变风格，增强视觉统一性。

### 5.3 交互效果

- 所有高亮单词在悬停时会切换渐变方向并增加亮度，提供反馈
- 弹窗在显示时应用淡入动画，增强视觉体验
- 所有交互元素都有平滑的过渡效果

## 6. 维护指南

### 6.1 新增词性

当需要支持新的词性时：
1. 在 `getPartOfSpeechClass` 函数中添加新的词性映射
2. 在CSS中添加对应的高亮和弹窗样式
3. 更新本样式规范文档

### 6.2 颜色调整

如需调整颜色方案：
1. 确保所有相关样式（高亮正常状态、高亮悬停状态、弹窗背景）保持一致的渐变风格
2. 更新本样式规范文档中的颜色值
3. 测试在不同背景下的可读性

### 6.3 性能优化

- 避免使用过多复杂的CSS渐变，保持样式简洁
- 利用CSS变量简化颜色管理（如后续引入）
- 定期审查和清理未使用的样式
